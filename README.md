# 포트폴리오 사이트 기능/UX 개선 기록 (Dobong-Practice)

## 1. 프로젝트 구조 (Repository Structure)

루트 폴더(최상위 폴더)명: dobong-portfolio-ux

* dobong-portfolio-ux/
    * README.md : 프로젝트 기록 및 명세서 (본 파일)
    * index.html : 단일 페이지 포트폴리오 (HTML/CSS/JS 통합)
        * images/

## 2. 기획 의도 (Why)

본 프로젝트는  자기소개(포트폴리오) 사이트를 단순히 "읽는 페이지"가 아니라, 사용자가 실제로 "사용하기 편한 페이지"로 만들기 위해 진행했습니다.

특히 AI를 활용하여 (Chat GPT) 다음 문제를 해결하는 것을 목표로 했습니다.

- 페이지가 보기에는 괜찮지만, 스크롤/네비게이션/FAQ 동작이 딱딱하게 느껴지는 문제
- 사이드바가 존재하지만 현재 위치 인지와 빠른 이동 역할이 약한 문제
- 스크롤스파이(active 표시)가 튀거나 오표시되는 문제
- Skills 상태(활용 가능/학습 예정)의 시각적 구분이 약한 문제
- 프로젝트가 없을 때 공백처럼 보이는 문제
- 정적(서버 없는) 환경에서 유지보수하기 쉬운 구조가 필요한 문제

결론적으로, 외부 라이브러리 없이 Vanilla JS로 UX 품질을 올리고, 단일 HTML 파일을 유지하며 확장 가능한 구조를 만드는 것을 목표로 했습니다.

## 3. 설계 및 구조 (Architecture)

1 레이아웃 구조
- 좌측 Sticky(고정) 사이드바 + 우측 메인 스크롤 구조로 확정했습니다.
- 사이드바는 CTA(버튼 중심) 역할이 아니라, 다음 역할로 정리했습니다.
  - 현재 위치 인지
  - 섹션 간 빠른 이동
  - 외부 링크(Repo/Demo 등) 접근의 단일 창구

2 반응형 구조
- 데스크톱: 2컬럼(좌측 고정 + 우측 스크롤)
- 모바일: 단일 컬럼으로 자연스럽게 전환되도록 유지했습니다.

3 섹션 구성
- Hero: 문장 중심으로 신뢰를 형성하는 구조(버튼 최소화)
- About: 배경 설명보다 일하는 방식이 바로 이해되도록 문구 직설화
- Skills: 활용 가능/학습 예정의 상태를 명확히 구분 + 필터링 제공
- Projects: 비어 있어도 UX가 깨지지 않도록 In Progress 상태로 처리
- FAQ: 클릭 동작이 부드럽게 느껴지도록 인터랙션 개선
- Contact: mailto 기반으로 단순화(정적 사이트 환경에 맞춤)

## 4. 핵심 로직 및 기술 선택 이유 (Core Logic & Selection)

1 사용 기술(현재 파일 기준)
- HTML/CSS/Vanilla JS 기반 단일 페이지
- 외부 라이브러리 의존성 없음
- 부드러운 스크롤(커스텀 이징)
- FAQ 열림/닫힘 부드러운 애니메이션(높이/opacity 기반)

2 기술 선택 이유
- 외부 라이브러리를 줄이고, 단일 파일에서 관리하기 위해 Vanilla JS를 선택했습니다.
- 단순 IntersectionObserver만 사용하면 active 표시가 튀거나 오표시되는 문제가 있어, 기준선(상단 1/3) 기반 스크롤스파이 로직으로 안정화했습니다.
- 네비게이션 클릭 시 기본 점프 동작은 사용자 경험이 딱딱하므로, 커스텀 이징을 적용한 부드러운 스크롤을 구현했습니다.
- FAQ는 기본 details/summary 동작이 "툭" 바뀌는 느낌이 강해, 열림/닫힘 애니메이션과 열린 항목 강조 스타일을 추가했습니다.

3 핵심 로직 개요
- 스크롤스파이
  - 섹션의 현재 위치를 기준선 기준으로 계산하여 active 상태를 업데이트했습니다.
  - 하단(Contact)에서 상단 섹션으로 잘못 고정되는 버그를 보정했습니다.
- 부드러운 섹션 이동
  - 네비게이션 클릭 시 목표 섹션 위치를 계산하고, offset을 적용해 보기 좋은 위치로 정렬했습니다.
  - requestAnimationFrame 기반 이징으로 자연스러운 이동을 구현했습니다.
- FAQ 모션
  - 열림/닫힘 시 answer 영역의 높이를 계산해 애니메이션으로 전환했습니다.
  - 열린 항목은 배경/테두리/그림자를 강화하여 상태 인지를 돕도록 했습니다.

## 5. 업데이트 기록 (Changelog)

v1.0 초기 구조 확인
- 1페이지 구조 확정(Hero/About/Skills/FAQ/Contact)
- 좌측 사이드바 + 우측 메인 레이아웃 기본 골격 구성

v1.1 레이아웃 구조 개선
- 사이드바 역할을 CTA가 아니라 현재 위치 인지 + 섹션 이동으로 정리
- 모바일에서 단일 컬럼 전환이 자연스럽게 유지되도록 구성 정리

v1.2 스크롤 기반 UX 개선
- 섹션 진입 reveal 애니메이션 적용
- 스크롤스파이 active 표시 기능 적용
- 튐/오표시 문제를 줄이기 위해 기준선(상단 1/3) 기반 로직으로 안정화
- Contact 구간에서 FAQ로 잘못 고정되는 문제 보정
- 빠른 스크롤 시 두 칸씩 튀는 현상 최소화

v1.3 네비게이션 클릭 UX 개선
- 사이드바 클릭 시 기본 점프 대신 부드러운 스크롤 적용(커스텀 이징)
- 섹션 상단 기준 위치로 오도록 scroll offset 적용(중앙 정렬 애매한 영역 제외)

v1.4 시각적 인터랙션 정리
- 마우스 커서 기반 subtle glow 적용
- glow 원 크기 과함 문제를 줄이기 위해 사이즈 조정
- hover 시 미세 이동 + 배경 변화로 클릭 가능 요소 인지 강화

v1.5 Skills 표현 강화
- Skills 필터 기능 유지(즉시 JS 필터링)
- 상태를 활용 가능/학습 예정 2개로 단순화
- 학습 예정 카드 톤다운으로 구분 강화
- 배지 2줄 깨짐 문제 해결(white-space: nowrap)

v1.6 프로젝트 없음 상태 처리
- Projects 공백 방지: In Progress / Coming Soon 상태 표시
- Demo/Repo 없음 상황에서 사용자 혼란이 없도록 안내 중심 구성

v1.7 Roadmap/Contact 현실화 및 단순화
- 교육 과정 중 과장된 로드맵 제거, "교육 종료 후 구체화"로 축소
- Contact를 mailto 기반으로 단순화(이메일 + 1줄 문구)
- 이메일 폰트 가독성 개선(불필요한 mono 사용 최소화)

v1.8 FAQ 인터랙션 개선(딱딱함 제거)
- FAQ 열림/닫힘을 부드러운 애니메이션으로 전환
- 열린 질문은 강조 배경/테두리/그림자 적용
- 아코디언 방식(한 번에 하나만 열리기) 지원

## 6. 실무 관점 (Insight)

1 구조 안정성
- 단일 파일 내에서 레이아웃/패턴/로직이 섞이면 유지보수가 어려워질 수 있으므로, 클래스/섹션 단위로 구조를 통일했습니다.
- 스크롤스파이는 "정확도"가 UX 체감에 큰 영향을 주기 때문에, 오표시를 줄이는 보정 로직이 중요하다고 판단했습니다.

2 데이터/상태의 명확성
- Skills는 "활용 가능/학습 예정"처럼 상태를 단순화하고, 시각적으로도 분리하여 사용자(면접관)가 한눈에 판단 가능하도록 했습니다.

3 사용자 관점의 불편 제거
- 네비게이션 클릭 시 점프는 사용자가 위치를 잃기 쉬우므로 부드러운 이동을 적용했습니다.
- FAQ는 기본 동작이 딱딱하여, 부드러운 전환과 열린 항목 강조로 사용성을 올렸습니다.
- Projects가 비어 있는 상황은 공백으로 두면 신뢰가 떨어질 수 있으므로, In Progress로 상태를 명확히 표현했습니다.

## 7. 성장 포인트 (Retrospective)

- 단순히 보기 좋은 UI보다, 스크롤/이동/상태 인지처럼 실제 사용 중 체감되는 UX 요소가 더 중요하다는 점을 체감했습니다.
- IntersectionObserver를 적용한다고 끝이 아니라, 실제 스크롤 상황(빠른 스크롤, 하단 구간, 섹션 높이 차이)에서 오표시가 발생할 수 있다는 점을 경험했습니다.
- 문구는 멋있어 보이는 표현보다, 읽자마자 이해되는 직설적인 표현이 더 신뢰를 만든다는 것을 느꼈습니다.
- 정적 환경에서는 기능을 늘리기보다, "혼란을 줄이는 설계"가 결과적으로 더 완성도를 올린다는 것을 배웠습니다.

## 기술 포인트 한 줄 요약
정적 페이지 포트폴리오를 스크롤스파이(정확도 보정), 부드러운 섹션 이동(커스텀 이징), FAQ 애니메이션, 상태 표현(활용 가능/학습 예정), 필터링, 미세 인터랙션으로 사용성 중심 개선을 진행했습니다.
